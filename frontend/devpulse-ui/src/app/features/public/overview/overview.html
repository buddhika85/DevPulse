DevPulse

A lightweight, end‑to‑end engineering productivity platform built with .NET, Azure, and Angular.

What DevPulse Is

DevPulse is a focused engineering productivity platform that helps developers and managers track work, surface blockers early, and maintain a healthy delivery rhythm. It brings together daily updates, progress visibility, and lightweight reporting into one simple, modern workflow.

Why I Built DevPulse

DevPulse began during a period where I was recovering, upskilling, and preparing for the next step in my career. I wanted to use that time intentionally — to deepen my knowledge of Azure, modern .NET, distributed systems, and CI/CD practices, while building something meaningful end‑to‑end.

Instead of learning these technologies in isolation, I decided to create a real product that would challenge me across architecture, cloud services, API design, containerization, and deployment pipelines. DevPulse became that project — a practical, hands‑on way to sharpen my engineering skills and build something I could confidently demo in interviews.

At the same time, I wanted DevPulse to solve a real problem: helping developers communicate progress clearly and helping managers understand team health without micromanagement. The result is a lightweight platform that reflects both my learning journey and my approach to building clean, reliable systems.

Architecture Overview

DevPulse is built as a lightweight, cloud‑native system following a clean, layered architecture. The goal was to design something simple enough to demo, yet structured enough to reflect real production patterns used in modern engineering teams.

The system is composed of multiple .NET microservices running in Azure Container Apps, fronted by an orchestrator service that acts as the backend‑for‑frontend (BFF) for the Angular UI. Azure API Management provides a secure and consistent entry point, while data is stored across SQL Database and Cosmos DB depending on access patterns.

This architecture allowed me to explore containerization, distributed system boundaries, API design, authentication flows, and CI/CD pipelines in a realistic but manageable way. Tech Stack

DevPulse is built using a modern, cloud‑ready stack designed for reliability, maintainability, and smooth developer experience. The backend follows clean architectural principles with clear separation of concerns, predictable domain boundaries, and lightweight orchestration.

  • Frontend: Angular 20, TypeScript, Angular Material, SCSS
  • Backend: ASP.NET Core 8 Web API with C#
  • Architecture Patterns: Domain‑Driven Design (DDD), CQRS with MediatR, Repository Pattern, Database‑per‑Service, Event‑Driven Messaging (Azure Service Bus)
  • Cloud: Azure Container Apps, Azure Functions, Azure SQL, Azure Blob Storage, Azure Container Registry, Azure Application Insights, Azure APIM, Azure Key Vault
  • Authentication: Microsoft Entra ID
  • DevOps: Azure DevOps Pipelines (CI/CD), Dockerisation of microservices, Azure Container Registry, Azure Container Apps
  • Database: SQL Server, EF Core (Code‑First + Migrations), Azure Cosmos DB
  • Testing: xUnit, Moq, FluentAssertions
  • Documentation & Tools: Swagger/OpenAPI, Postman, Git

CI/CD Pipeline & Automated Testing

DevPulse uses a fully automated CI/CD pipeline built with Azure DevOps. Every commit triggers a clean build, runs the full test suite, creates Docker images for each microservice, and pushes them to Azure Container Registry. Successful builds are automatically deployed to Azure Container Apps, ensuring fast and reliable delivery.

The solution includes meaningful unit tests using xUnit, Moq, and FluentAssertions. These tests validate domain logic, service orchestration, error handling, and repository behavior. Test results are surfaced directly in the pipeline, providing immediate feedback and preventing regressions.

Azure DevOps pipeline runs unit tests on every commit, publishing results to the Tests tab and enforcing quality gates before deployment.

Vision

DevPulse began during a period where I was recovering, upskilling, and preparing for the next step in my career. I wanted to use that time intentionally — to deepen my knowledge of Azure, modern .NET, distributed systems, and CI/CD practices, while building something meaningful end‑to‑end.

Over time, that personal learning project evolved into a long‑term vision: a lightweight engineering performance platform that helps developers understand their work patterns, track meaningful progress, and stay aligned with their goals without adding friction to their day.

The roadmap focuses on expanding DevPulse into a modular, cloud‑native system that supports richer insights, team‑level visibility, and seamless integrations with the tools developers already use. Each iteration aims to improve clarity, reduce noise, and make engineering work feel more intentional and measurable.

Planned Areas of Expansion

  • Team Dashboard: Aggregate developer signals to give engineering managers a clear, respectful, and data‑driven view of team health and progress.
  • AI‑Assisted Insights: Use patterns from journals, tasks, and mood entries to surface trends, blockers, and opportunities for improvement.
  • Calendar & Worklog Integration: Connect with Outlook, GitHub, and Azure DevOps to automatically enrich daily entries.
  • Personal Growth Metrics: Track consistency, focus time, learning habits, and long‑term progress across weeks and months.
  • Mobile App: A lightweight companion for quick journaling and mood tracking on the go.

The goal is simple: help developers build better habits, understand their own patterns, and create a healthier, more intentional engineering culture — one entry at a time.

Built with intention, curiosity, and a deep respect for the craft of engineering. DevPulse is my way of learning out loud — one commit, one insight, one improvement at a time.
